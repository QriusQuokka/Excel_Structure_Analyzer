# Excel Structure Analyzer

처음 받아본, 설명해 줄 사람도 없는 다중 시트 엑셀 파일의 **구조와 시트 간 수식 관계**를
자동으로 파악해 한 장의 HTML 리포트로 보여주는 도구입니다.

- 모든 시트 목록과 크기·수식 개수 파악
- 시트별 수식 추출
- 수식 안의 시트 참조(`SheetB!A1`, `'시트 이름'!A1`, 3D 참조 `Sheet1:Sheet3!A1` 등)를 분석해
  **시트 간 데이터 흐름(입력 → 가공 → 산출)** 을 인터랙티브 그래프로 시각화
- **입력단/산출단 시트 자동 식별**, 순환 참조 감지

분석 엔진은 하나(`excel_analyzer/`)이고, **웹**과 **CLI** 두 가지 방법으로 쓸 수 있습니다.

## 🔒 프라이버시

웹으로 쓸 때 **파일은 브라우저 밖으로 나가지 않습니다.** 서버 업로드 없이 분석이 전부
브라우저 안(Pyodide = 브라우저에서 도는 Python)에서 이뤄지고, 결과 HTML·ZIP도 그 자리에서
만들어 내려받습니다. 회사 기밀 엑셀도 안심하고 분석할 수 있습니다.

## 사용법 1 — 웹 (설치 불필요)

브라우저로 페이지를 열고 엑셀 파일을 끌어다 놓으면 됩니다.

- 여러 파일을 한 번에 올려 일괄 분석(파일별 리포트 탭 전환)
- 결과를 **개별 HTML** 또는 **전체 ZIP**으로 다운로드
- 내려받은 HTML은 인터넷 없이 열어도 그래프가 동작

**로컬에서 바로 띄우기** (※ `index.html`을 파일로 직접 열면 보안정책상 동작하지 않으니 서버로 여세요):

```bash
python -m http.server 8765
# 브라우저에서 http://localhost:8765 접속
```

**바로 사용**: 👉 **<https://qriusquokka.github.io/Excel_Structure_Analyzer/>**
(첫 접속 때만 런타임 ~10MB를 받아 준비하는 데 약 10초가 걸립니다.)

## 사용법 2 — CLI

Python 3.10+ 와 openpyxl 만 있으면 됩니다.

```bash
pip install -r requirements.txt        # 또는: pip install openpyxl
python analyze.py 파일.xlsx
```

옵션:

| 옵션 | 설명 |
|------|------|
| `-o, --output 경로` | 출력 HTML 경로 (기본: `입력파일명_analysis.html`) |
| `--no-open` | 분석 후 브라우저를 자동으로 열지 않음 |

실행하면 HTML 리포트가 생성되고 브라우저가 자동으로 열립니다.
검증용 샘플은 `python make_sample.py` 로 `sample.xlsx` 를 만들어 쓸 수 있습니다.

## 리포트 보는 법

- **상단 통계 칩**: 시트 수 / 총 수식 수 / 시트 간 참조 수 / 순환 참조 유무
- **시트 간 관계도**: 사각형 노드(시트)를 드래그해 배치를 바꿀 수 있습니다.
  화살표는 데이터 흐름 방향이며, `A → B` 는 "B가 A의 값을 참조한다"는 뜻입니다.
  노드 색은 시트의 **역할**을 의미합니다:
  - **입력단**(청록) — 다른 시트를 참조하지 않는, 데이터가 들어오는 시트
  - **가공**(회색) — 참조도 하고 참조도 당하는 중간 시트
  - **산출단**(그린) — 아무 시트도 참조하지 않는, 보통 최종 결과 시트
  - **독립**(옅은 회색) — 시트 간 참조가 없는 시트
- **시트 목록**: 각 시트의 사용 범위·크기·수식 수·숫자/텍스트 셀·수식 비율·참조하는 시트
- **시트별 수식 상세**: 시트 기준 전체 수식을 접철식으로 확인

## ⚠️ 구버전 .xls 파일

이 도구는 `.xlsx` / `.xlsm` 만 분석합니다.
오래된 **`.xls` 파일은 엑셀에서 열어 [다른 이름으로 저장] → `Excel 통합 문서(*.xlsx)`** 로
다시 저장한 뒤 그 파일을 분석해 주세요. (.xls 를 넣으면 같은 안내 메시지가 나옵니다.)

## 한계

- 시트 단위 관계에 초점을 맞춥니다(셀 단위 추적은 "시트별 수식 상세"에서 확인).
- 외부 통합문서(`[1]Sheet!A1`) 참조나 이름 정의(Named Range)를 통한 참조는 관계도에
  표시되지 않으며, 외부 참조 개수는 리포트 하단 "참고"에 별도 표시됩니다.
- 웹은 첫 접속 시 런타임을 받기 위해 인터넷이 필요합니다(이후 분석 자체는 오프라인 동작).
- 여러 파일을 올려도 **파일 간**(A.xlsx ↔ B.xlsx) 관계는 분석하지 않습니다 — 각 파일을
  개별 리포트로 분석합니다.

## 구조

```
excel_analyzer/
  __init__.py         공개 API (analyze_workbook, WorkbookAnalysis, SheetInfo, Dependency)
  formula_parser.py   수식에서 시트 참조 추출 (정규식 + 실제 시트명 대조)
  analyzer.py         openpyxl 로 시트/수식/의존관계/순환참조 분석
  report.py           분석 결과 → 단일 HTML 리포트 (관계도는 순수 Canvas + JS, 외부 의존 없음)
analyze.py            CLI 진입점
make_sample.py        검증용 샘플 통합문서 생성
index.html, app.js    웹 UI (Pyodide 로 같은 엔진을 브라우저에서 실행)
```
